# GitHub Flow — In My Own Words

1. **Branch off main** — create a new branch for the specific change or feature so main stays stable.
2. **Make commits** — work on the branch, committing changes with clear, descriptive messages as you go.
3. **Push the branch** — push the branch to GitHub so the work is backed up and visible to others.
4. **Open a pull request** — describe what the branch changes and why, so others can review it.
5. **Review and discuss** — collaborators comment on the PR, request changes, or approve it.
6. **Merge into main** — once approved (and checks pass), merge the PR into the main branch.
7. **Clean up** — delete the merged branch and pull the updated main locally to stay in sync.
